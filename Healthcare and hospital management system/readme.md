# Healthcare Medallion Architecture — Reference Implementation

A working PySpark implementation of a Bronze → Silver → Gold pipeline for hospital
data: 5 CSV sources, metadata-driven ingestion, full audit logging, HIPAA-aligned
PII masking, and 8 KPIs with GREEN/AMBER/RED alerting.

## Quick start

```bash
pip install pyspark==3.5.1 --break-system-packages
python3 generate_source_data.py        # generates 5 synthetic source CSVs (~4,700 rows)
python3 notebooks/08_run_pipeline.py   # runs all 8 notebooks end to end
```

Each notebook also runs standalone, e.g. `python3 notebooks/06_gold_kpis.py`.

## Project layout

```
utils/spark_utils.py           # Spark session, table I/O, audit-log schema (shared)
notebooks/
  00_setup_config.py           # metadata config table + audit log init
  01_ingest_bronze.py          # metadata-driven ingest, all 5 sources, retries, SLA checks
  02_silver_patients.py        # dedup, DQ scoring, SCD Type 2, SHA-256 PII masking
  03_silver_appointments.py    # date validation, no-show flag, LOS calc, patient join
  04_silver_billing.py         # charge validation, billing-accuracy / claim-approval flags
  05_silver_lab_meds.py        # abnormal-flag cross-check, inventory/turnover prep
  06_gold_kpis.py              # all 8 KPIs + thresholds + per-department breakdowns
  07_audit_report.py           # pipeline run summary, failures, SLA, DQ trend
  08_run_pipeline.py           # orchestrator (runs 00–07 in order)
data/
  landing/    bronze/    silver/    gold/    audit/    config/    quarantine/
reports/audit_report.md        # generated by notebook 07
```

## What's implemented

- **Metadata-driven ingestion** — one notebook loads all 5 sources off a config table (20 columns), with retry logic, row-count/null-pct thresholds, and `last_load_timestamp` tracking
- **Full audit trail** — 24-column audit log, one row per source per layer per run
- **Data quality** — null/dedup checks, DQ scoring, failed rows written to `quarantine_path`
- **SCD Type 2 + PII masking** — SSN/DOB/phone SHA-256-hashed and dropped in Silver
- **8 Gold KPIs** with real thresholds (no-show rate, ALOS, HCAHPS score, billing accuracy, claim approval rate, medication turnover, cost per patient, readmission rate), each with its own table plus a summary table
- **HIPAA controls**: PII masking, immutable audit log, no PHI in Gold, data classification tagging

## Known limitations

- **Delta Lake → Parquet stand-in.** Target platform is Databricks + Delta Lake; this build
  uses Parquet locally (no Maven access in this environment). Change `FORMAT = "parquet"` to
  `"delta"` in `utils/spark_utils.py` to restore true Delta semantics on Databricks.
- **No `MERGE INTO`.** SCD Type 2 currently overwrites with fresh "current" versions each run
  rather than doing an incremental merge — needs Delta Lake's `MERGE INTO` on a real workspace.
- **Not simulated** (require an actual Azure/Databricks deployment): Unity Catalog RBAC,
  column-level security, ADLS encryption at rest/in transit, Databricks Workflows scheduling,
  Great Expectations integration, and real email/webhook notifications.
